	Функции, которые планируется использовать извне, требуется экспортировать. Например, extern "C".
	Библиотеки импорта - статические библиотеки, которые компонуются с помощью компоновщика. В статической библиотеке находятся функции с такими же именами, как экспортируемые в DLL. Они загружают нужную библиотеку. Библиотеки импорта создаются компоновщиком при подключении DLL.
	Спецификация COM (Common Object Model) - спецификация для разработки приложений, состоящих из слабосвязанных компонентов. Может использоваться в любой ОС, но прижилась только в Windows. Задача - представление универсальных компонентов, которые стыковались бы между собой. COM-программирование - разработка программных объектов времени исполнения (ранее мы разрабатывали объекты времени компиляции).
	GUID (Global Unique Identifier) - 128-битный идентификатор.
	COM-объекты могут размещаться в DLL и EXE.
	Статическая область и область кода остаётся общей для всех, область данных и стек свои для каждого процесса. Такой разнос секций называется проецированием (...DLL).
	библиотека OLE32. Заголовок <objbase.h>.
	CLSCTX_INPROC_SERVER (DLL внутрипроцессный сервер), CLSCTX_LOCAL_SERVER (EXE-сервер за границами процесса, но на том же компьютере), CLSCTX_REMOTE_SERVER (EXE-сервер на удалённом компьютере).
	COM-клиент - программа, обращающаяся к COM-серверу, вызывающая функцию OLE32 и создающая COM-объект, с которым дальше начинает работать. COM-клиент -> вызов функции-> OLE32 -> вызов функции -> СОМ-сервер. СОМ-сервер -> возврат управления -> OLE32 -> возврат управления -> СОМ-клиент.
	QueryInterface нужен, чтобы получить указатель другого интерфейса. В качестве параметра передаётся другой интрефес, а получается адрес
	Чтобы  COM-клиент мог работать с COM-сервером, он должен знать CLSID, тип сервера и интерфейсы
	Интерфесы делятся на две группы: создающиеся своиими руками и стандартные (у которых известен идентификатор и этот идентификатор вем известен). Например, IUnknown (особый интерфейс, от него должен наследоваться любой интерфейс в СОМ-компоненте).
