	Если программа пишется на уровне api ОС и она жёстко привязана к железу, то такое программирование называется системным (разработка ОС, компиляторов, СУБД)
	Программирование для других программистов
	Задача системного программиста - создать новый уровень абстракции для другого программиста, чтобы скрыть детали того, что на самом деле происходит
	В состав любой ОС входит специальный набор библиотек (фреймворк операционной системы)
	Фреймворк - набор библиотек и правила их применения
	В рамках ОС всё разрабатывается при помощи фреймворка ОС
	POSIX - стандарт операционных систем (сначала unix, затем linux), в котором описывается, как программа, работающая в рамках ОС, может взаимодействовать с ОС
	Нет никакого другого способа что-либо сделать с компьютером, кроме вызова API ОС
	Ядро - набор функций, которые постоянно находятся в ОЗУ, и чтобы добраться до этих функций нужно использовать механизм системных вызовов
	Системные вызовы могут быть разными: выполняемыми программистом, выполняемые оборудованием. В обоих случаях используется механизм системного прерывания (прерывается текущее исполнение кода и вызывается функция ядра ОС, а после её выполнения исполнение кода продолжается)
	Функции API ОС внутри себя содержат специальный прораммный код, который осуществляет системный вызов
	В арбитр собираются сигналы от разной аппаратуры и при одновременном приходе сигналов арбитр выбирает, кому отдать приоритет
	API ОС делает системные вызовы
	Процесс - 1) капусла, которая изолирует исполняемый код; 2) объект ядра ОС, в ядре есть специальные функции и специальный системный вызов, создающий процесс (функция createProcess)
	Поток - специальный механизм, который предназначен для исполнения команд процесса. Поток выделяется в рамках процесса. Поток - набор команд, исполняемых процессором. При создании потока указывается функция, которую будет выполнять поток. В прцоессе всегда существует главный поток
	Размер стека и кучи определяется компоновщиком. Второй способ выделения памяти - виртуальная память
