1. Что такое MOM, JMS?
МОМ - вспомогательное программное обеспечение для работы с сообщениями, обычно входящее в поставку сервера приложений
JMS - API с помощью которого Java программа может оправлять сообщения(объекты!!)

2. Поясните принцип работы MOM. Варианты архитектуры.
MOM создает распределенный уровень связи, который изолирует разработчика приложения от деталей различных операционных систем и сетевых интерфейсов. MOM обычно предоставляет API, которые расширяются на различные платформы и сети.
Типы: RPC-ориентированная (синхронная), сообщение-ориентированная

3. Объясните принцип работы режима point-to-point (P2P).
сообщение, которое отправил producer, получает единственный consumer

4. Объясните принцип работы publish/subscribe (pub/sub).
одно сообщение может быть прочитано неограниченным количеством consumer, подписанных на этот destination

8. Охарактеризуйте администрируемые объекты JMS.
Типы объектов:
► фабрики соединений — используются клиентами для создания подключения к пункту назначения;
► места назначения — точки распространения сообщений, которые получают, хранят и распространяют сообщения; (P2P) (pubsub).

9. Структура сообщения JMS.
- header - unique message ID, destination, type и т.д. – заполняется автоматически при send() или publish(), клиентом и поставщиком
- доп. свойства (опц.) – запол. программно
- body (опц.) – содержит сообщение для доставки

10.Типы сообщений JMS.
►TextMessage - объект с Java String
►ObjectMessage - сериализуемые Java объекты
►BytesMessage - массив байт
►StreamMessage - поток Java примитивов
►MapMessage - key/value пары
