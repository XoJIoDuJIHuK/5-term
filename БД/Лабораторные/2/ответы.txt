1.Перечислите редакции поставки Oracle 12с.
	standart edition one, two, enterprise, exress, personal
2.Перечислите предопределенных пользователей БД Oracle.
	sys (суперпользователь), system (базовый пользователь базы данных)
3.Что такое табличное пространство?
	Табличное пространство - это логическое пространство, которое используется для хранения объектов базы данных, таких как таблицы и индексы
4.В каком соотношении находятся табличные пространства и файлы данных?
	Табличные пространства и файлы данных находятся в соотношении один ко многим. Табличное пространство может содержать один или несколько файлов данных
5.Перечислите известные вам параметры табличного пространства.
	DATAFILE - файл ОС;
	SIZE - размер табличного пространства (начальный);
	AUTOEXTEND ON NEXT - приращение;
	MAXSIZE - максимальный размер
6.Перечислите типы табличных пространств и их назначение.
	PERMANENT - предназначены для хранения постоянных объектов (таблиц, индексов, кластеров, ...)
	TEMP - используется для временных объектов;
	UNDO - используется для отката
7.Что такое роль и для чего она применяется?
	именованный набор привелегий
8.Какие системные роли вы знаете? Объясните их назначение.
	CONNECT - позволяет пользователю подключаться к базе данных;
	RESOURCE - позволяет пользователю создавать объекты базы данных;
	DBA - позволяет пользователю выполнять любые операции в базе данных
9.Что такое системная привилегия, какие системные привилегии вы знаете?
	право на выполнение определенной операции в базе данных.
	CREATE TABLESPACE - позволяет пользователю создавать табличные пространства; 
	CREATE SESSION - позволяет пользователю подключаться к базе данных; 
	CREATE USER - позволяет пользователю создавать новых пользователей
10.Какие специальные системные привилегии администратора БД вы знаете? Как их назначить?
	SYSDBA - позволяет администратору выполнять любые операции в базе данных;
	SYSOPER - позволяет администратору выполнять некоторые операции в базе данных, такие как запуск и остановка экземпляра базы данных
	Назначить их можно пользователям, созданным без аутентификации
11.Что такое профиль безопасности и для чего он нужен?
	Профиль безопасности - набор ограничений на использование ресурсов базы данных для пользователя или роли
12.Перечислите известные вам параметры профиля безопасности и поясните их.
	PASSWORD_LIFE_TIME: количество дней жизни пароля
	SESSIONS_PER_USER: количество сессий на одного пользователя
	PASSWORD_LOCK_TIME: количество дней блокировок после ошибок
	FAILED_LOGIN_ATTEMPTS: Количество неудачных попыток входа, прежде чем учетная запись будет заблокирована. По умолчанию равна трем попыткам
	PASSWORD_GRACE_TIME: Период действия пароля, когда значение параметра PASSWORD_LIFE_TIME превышено
	PASSWORD_LIFE_TIME: Указывает, как долго действует пароль
	PASSWORD_REUSE_TIME: через сколько дней можно повторить пароль
	CONNECT_TIME: время соединения в минутах
	IDLE_TIME: количество минут простоя
13.Объясните назначение профиля с именем DEFAULT.
	используется по умолчанию для всех пользователей, которым не назначен другой профиль безопасности
14.Что такое пользователь базы данных?
	учетная запись, которая используется для подключения к базе данных Oracle
15.Перечислите известные вам параметры пользователя БД Oracle.
	USERNAME - имя пользователя;
	ACCOUNT_STATUS - статус учетной записи;
	DEFAULT_TABLESPACE - табличное пространство по умолчанию для пользователя
16.Что такое квота?
	ограничение на использование ресурсов табличного пространства для пользователя или роли
17.Перечислите все известные вам представления словаря БД Oracle.
	DBA_ROLES - детализирует все роли, хранящиеся в базе данных
	DBA_USERS - содержит информацию о всех пользователях базы данных;
	DBA_PROFILES - для каждого профиля хранит информацию о ресурсах и их лимитах
	DBA_TABLESPACES - описание всех табличных пространств
	USER_ROLE_PRIVS - роли, назваченные текущему пользователю

Запросы к словарю данных
- USER	Объекты, принадлежащие пользователю
- ALL	Объекты, к которым пользователь имеет доступ 
- DBA	Все объекты базы данных (для администратора БД)
- V$	Производительность сервера
