30. Что означает свойство табличных пространств smallfile/bigfile?
	smallfile – 1022 файлов, по 222 блоков. bigfile – 1 файл, 128TB(блок 32К) или 32TB(блок 8К)

31. Что означает свойство табличных пространств logging/nologging?
	Свойство табличных пространств logging/nologging в Oracle 12c относится к тому, генерируются ли записи журнала redo для операций с объектами.

32. Что означает свойство табличных пространств offline/online?
	Когда табличное пространство установлено в состояние offline в Oracle 12c, большинство операций с данными становятся недоступными. В частности, вы не сможете выполнять следующие операции:
	Чтение данных: Пользователи не могут читать данные из таблиц, которые находятся в табличном пространстве.
	Запись данных: Пользователи не могут вставлять, обновлять или удалять данные из таблиц, которые находятся в табличном пространстве.
	Создание или удаление объектов: Вы не сможете создавать или удалять таблицы или другие объекты в табличном пространстве.

33. Каким образом можно выяснить наименование применяемого инстансом Oracle 12c имя UNDO-табличного пространства?
	Указывается в файле параметров экземпляра (*.undo_tablespace='UNDOTBS1')

34. Что такое сегмент табличного пространства?
	Сегмент – область на диске, выделяемая под объекты. Сегменты типизируются  в зависимости от типа данных, хранящихся в них – сегменты таблиц, сегменты индексов, сегменты кластеров и т.д.(всего 10 типов).

35. Перечислите типы сегментов? Как получить все типы сегментов?
	Таблиц, индексов, кластеров, данных, отката, временные

36. Удаляется ли (или сокращается) сегмент таблицы при удалении (DELETE) всех строк таблицы? 
	сегмент не удаляется, он продолжает ассоциироваться с таблицей

37. Что происходит с сегментом таблицы XXX_T1 при удалении ее оператором drop table XXX_T1?
	При использовании оператора DROP TABLE для удаления таблицы в Oracle 19c, сегмент таблицы удаляется, и все связанные с ним данные, индексы, триггеры и ограничения также удаляются. Однако, таблица и её сегмент не удаляются полностью, а перемещаются в корзину для восстановления (recycle bin).

38. Поясните назначение представление USER_RECYCLEBIN.
	Представление USER_RECYCLEBIN в Oracle 19c отображает информацию о корзине для восстановления (recycle bin), принадлежащей текущему пользователю. Это представление содержит информацию о всех объектах, которые были удалены оператором DROP, но которые еще можно восстановить с помощью оператора FLASHBACK.

39. Что происходит с сегментом таблицы XXX_T1 при удалении ее оператором drop table XXX_T1 purge?
	Если вы используете оператор DROP TABLE с ключевым словом PURGE для удаления таблицы в Oracle 19c, сегмент таблицы полностью удаляется из базы данных и не перемещается в корзину для восстановления

40. Что такое экстент табличного пространства?
	Экстент – непрерывный фрагмент дисковой памяти. Является единицей выделения вторичной памяти (выделяется целым числом экстентов). Когда экстент заполняется выделяется следующий. Размер экстента варьируется от одного блока до 2 Гб.

41. Поясните назначение опции EXTENT MANAGEMENT LOCAL при создании табличного пространства.
	Опция EXTENT MANAGEMENT LOCAL при создании табличного пространства в Oracle 19c указывает на то, что управление экстентами будет осуществляться на уровне каждого отдельного табличного пространства. Это означает, что информация об экстентах хранится в заголовках данных каждого табличного пространства, а не в словаре данных.

42. Поясните назначение опции UNIFORM при создании табличного пространства.
	Опция UNIFORM при создании табличного пространства в Oracle 19c используется для указания, что все экстенты в табличном пространстве будут одинакового размера. Это может быть полезно, если ожидается, что табличное пространство будет содержать объекты одинакового размера, требующие экстентов одинакового размера.

43. Что такое блок данных табличного пространства?  Где и как задается его размер? Как выяснить размер блока? 
	Блок – минимальная единица объема памяти, применяемая при записи и чтении данных. Размер кратен 2К, и должен быть кратен величине блока операционной системы (2К, 4К, 8К, допустимы 16К, 32К). Устанавливается в файле параметров экземпляра при создании БД. В табличном пространстве все блоки одного размера.

54. Что такое файл параметров? Как выяснить его местоположение? В чем разница между SPFILE  и PFILE? Какая возможность появляется (приведите пример) при наличии бинарного формата файла параметров? 
	Файл параметров предназначен для хранения параметров экземпляра. Месторасположение файла параметров по умолчанию - ROACLE_HOME\database
	server parameter file - двоичный, при его изменении необязательно перезапускать инстанс
	parameter file - текстовый, можно менять вручную
	ALTER SYSTEM SET SGA_TARGET = 500M SCOPE = BOTH; Эта команда изменит значение SGA_TARGET в текущем экземпляре базы данных и обновит SPFILE, чтобы изменения были постоянными.

55. В какой последовательности инстанс ищет файлы параметров?
	SPFILE (Server Parameter File): Oracle сначала ищет бинарный файл параметров SPFILE. Обычно он называется spfile<SID>.ora, где <SID> - это идентификатор системы базы данных.
	PFILE (Parameter File): Если Oracle не может найти SPFILE, он затем ищет текстовый файл параметров PFILE. Обычно он называется init<SID>.ora.
	Оба этих файла обычно находятся в %ORACLE_HOME%\database на Windows.	

56. Каким образом можно получить PFILE из SPFILE? Каким образом можно получить SPFILE из PFILE?
	create pfile = 'p1.ora' from spfile
	create spfile = 'sp1.ora' from pfile = 'p1.ora'

	%ORACLE_HOME%\database, PWD<sid>.ora
